================================================================================
          DEPLOYMENT GUIDE - LinkedIn Job Automation with AI
================================================================================

This guide will walk you through deploying the LinkedIn Job Automation
system to production using Render (Backend) and Vercel (Frontend).

================================================================================
                        PREREQUISITES
================================================================================

Before you begin, ensure you have:

1. ✓ GitHub Account (to push your code)
2. ✓ Render Account (https://render.com - Free to sign up)
3. ✓ Vercel Account (https://vercel.com - Free to sign up)
4. ✓ Supabase Account (https://supabase.com - For PostgreSQL database)
5. ✓ OpenAI API Key (https://platform.openai.com)
6. ✓ Google Gemini API Key (https://ai.google.dev) - Optional
7. ✓ LinkedIn Account (for automation)
8. ✓ Git installed on your local machine


================================================================================
                        PHASE 1: REPOSITORY SETUP
================================================================================

Step 1: Push Code to GitHub
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Open terminal in project directory:
   
   cd d:\Projects\LinkedIn-Job-Automation-with-AI

2. Initialize Git (if not already):
   
   git init

3. Create .gitignore file (ensure sensitive files are excluded):
   
   The project should already have a .gitignore. Verify it includes:
   - .env
   - __pycache__/
   - *.pyc
   - node_modules/
   - uploads/
   - data/logs/

4. Add all files:
   
   git add .

5. Commit:
   
   git commit -m "Initial commit for deployment"

6. Create a new repository on GitHub:
   - Go to https://github.com/new
   - Name: linkedin-job-automation-ai (or your preferred name)
   - Make it Public or Private
   - DO NOT initialize with README (you already have one)
   - Click "Create repository"

7. Add remote and push:
   
   git remote add origin https://github.com/YOUR_USERNAME/linkedin-job-automation-ai.git
   git branch -M main
   git push -u origin main

✓ Your code is now on GitHub!


================================================================================
                        PHASE 2: DATABASE SETUP (SUPABASE)
================================================================================

Step 2: Create Supabase Project
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Go to https://supabase.com

2. Click "Start your project"

3. Sign up or log in (GitHub OAuth is easiest)

4. Click "New Project"

5. Fill in details:
   - Name: linkedin-automation (or your choice)
   - Database Password: Create a STRONG password (save this!)
   - Region: Choose closest to your Render region (e.g., US East for Oregon Render)
   - Pricing Plan: Free tier is fine to start

6. Click "Create new project" - Wait 2-3 minutes for provisioning

7. Once ready, go to "Project Settings" → "Database"

8. Copy these values (you'll need them later):
   
   ┌────────────────────────────────────────────────────────────────┐
   │ IMPORTANT: Save these values securely!                         │
   ├────────────────────────────────────────────────────────────────┤
   │ DATABASE_URL (Connection string):                              │
   │   postgresql://postgres:[PASSWORD]@[HOST]:5432/postgres        │
   │                                                                │
   │ SUPABASE_URL:                                                  │
   │   https://xxxxx.supabase.co                                    │
   │                                                                │
   │ SUPABASE_KEY (anon/public):                                    │
   │   eyJhbG...                                                    │
   └────────────────────────────────────────────────────────────────┘

9. Initialize Database Schema:

   Option A - Using Supabase SQL Editor:
   - Go to "SQL Editor" in Supabase dashboard
   - Click "New query"
   - Copy the contents from: database/init.sql
   - Paste and click "Run"

   Option B - Using local script (if you have psql):
   - Install PostgreSQL client if needed
   - Run: psql "postgresql://postgres:[PASSWORD]@[HOST]:5432/postgres" -f database/init.sql

✓ Database is now set up!


================================================================================
                        PHASE 3: API KEYS SETUP
================================================================================

Step 3: Obtain Required API Keys
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

A. OpenAI API Key (Required)
   
   1. Go to https://platform.openai.com
   2. Sign up / Log in
   3. Go to "API keys" section
   4. Click "Create new secret key"
   5. Name it: "LinkedIn Automation"
   6. Copy the key (starts with sk-...)
   7. Save it securely - you can't see it again!
   
   Cost: Pay-as-you-go, ~$10-50/month depending on usage

B. Google Gemini API Key (Optional, but recommended for free tier)
   
   1. Go to https://ai.google.dev
   2. Click "Get API key in Google AI Studio"
   3. Sign in with Google account
   4. Click "Create API key"
   5. Copy the key
   
   Cost: Free tier - 60 requests/minute

C. Generate a Secret Key (for JWT)
   
   On Mac/Linux:
   openssl rand -hex 32
   
   On Windows PowerShell:
   -join ((48..57) + (65..90) + (97..122) | Get-Random -Count 32 | % {[char]$_})
   
   Or use an online generator: https://randomkeygen.com/
   Choose "CodeIgniter Encryption Keys" (256-bit)

D. Google OAuth Client ID (Optional - for social login)
   
   1. Go to https://console.cloud.google.com
   2. Create new project or select existing
   3. Enable "Google+ API"
   4. Go to "Credentials" → "Create Credentials" → "OAuth client ID"
   5. Application type: Web application
   6. Authorized JavaScript origins:
      - http://localhost:8080 (for development)
      - https://your-app.vercel.app (add after Vercel deployment)
   7. Copy Client ID

✓ API keys are ready!

================================================================================
                        TROUBLESHOOTING
================================================================================

Common Deployment Issues & Fixes
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Issue 1: Render build fails with "Playwright installation error"
Fix:
  - Check build.sh has execute permissions
  - Verify render.yaml has correct build command
  - Increase instance size (Free tier may be too small)
  - Check build logs for specific error

Issue 2: Backend returns 500 Internal Server Error
Fix:
  - Check Render logs for Python errors
  - Verify all environment variables are set
  - Test DATABASE_URL connection
  - Ensure SECRET_KEY is set
  - Check PYTHONPATH in start command

Issue 3: Frontend shows "Network Error" or "CORS Error"
Fix:
  - Verify VITE_API_URL is correct (with https://)
  - Check CORS_ORIGINS in Render includes Vercel URL
  - Ensure no trailing slashes in URLs
  - Open browser console to see exact error
  - Try adding frontend URL to CORS_ORIGINS

Issue 4: Database connection fails
Fix:
  - Verify DATABASE_URL format is correct
  - Check Supabase project is not paused (free tier)
  - Test connection from Render shell:
    - Go to service → Shell
    - Run: python3
    - import os; print(os.environ.get('DATABASE_URL'))
  - Ensure password has no special URL characters (URL encode if needed)

Issue 5: Playwright browser crashes
Fix:
  - Upgrade Render instance (need at least 512 MB RAM)
  - Check PLAYWRIGHT_BROWSERS_PATH is set
  - Verify Chromium installed successfully in build logs
  - Set PLAYWRIGHT_HEADLESS=true

Issue 6: LinkedIn login fails in production
Fix:
  - Verify LINKEDIN_EMAIL and LINKEDIN_PASSWORD are correct
  - Check if LinkedIn flagged the login as suspicious
  - Try logging in manually to your LinkedIn account
  - Disable 2FA temporarily for testing
  - Reduce automation frequency
  - Check browser screenshots in logs

Issue 7: Vercel build fails
Fix:
  - Verify Root Directory is set to: frontend/lovable
  - Check package.json exists in frontend/lovable
  - Ensure all dependencies are in package.json
  - Check Build Command: npm run build
  - Verify Output Directory: dist
  - Look at detailed build logs

Issue 8: Environment variables not loading
Fix:
  - Ensure variable names are EXACTLY correct (case-sensitive)
  - No spaces around = sign
  - For Vercel, must start with VITE_ to be exposed to frontend
  - Redeploy after adding new variables
  - Check spelling

Issue 9: Application submissions fail
Fix:
  - Start with dry_run=true to debug
  - Check LinkedIn hasn't changed their form structure
  - Verify resume file uploads correctly
  - Check browser screenshots for captcha
  - Reduce max_applications to prevent rate limiting
  - Review error logs in applications table

Issue 10: High costs / API rate limits
Fix:
  - Enable response caching for LLM calls
  - Switch to Gemini (free tier) from OpenAI
  - Reduce max_applications per day
  - Implement request throttling
  - Monitor usage in OpenAI dashboard

Updating Your Deployment
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

To deploy updates:

1. Make changes to your code locally

2. Test locally:
   - Backend: PYTHONPATH=$PWD uvicorn backend.main:app --reload
   - Frontend: cd frontend/lovable && npm run dev

3. Commit changes:
   git add .
   git commit -m "Describe your changes"

4. Push to GitHub:
   git push origin main

5. Automatic deployment:
   - Render: Auto-deploys backend (if auto-deploy enabled)
   - Vercel: Auto-deploys frontend (always automatic)

6. Monitor deployment:
   - Render: Watch logs in dashboard
   - Vercel: Check deployment status

7. Smoke test after deployment:
   - Test key endpoints
   - Verify no breaking changes

Tips to Reduce Costs
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Use Free Tiers:
   - Vercel: Free for personal projects
   - Supabase: 500 MB database free
   - Gemini AI: 60 requests/min free
   - Total: $7-25/month (only Render backend)

2. Optimize AI Usage:
   - Cache LLM responses for common questions
   - Use Gemini instead of GPT-4 (free tier)
   - Batch embedding generation
   - Reduce token count in prompts

3. Database Optimization:
   - Clean up old job listings
   - Archive old applications
   - Use indexes for common queries
   - Stay within free tier limits

4. Backend Optimization:
   - Use smaller Render instance if possible
   - Enable compression (GZip)
   - Implement caching
   - Reduce log verbosity

5. Monitoring:
   - Set up budget alerts
   - Track API usage daily
   - Monitor Render instance metrics
   - Review costs monthly
